# Overview

跨编译单元的静态/全局变量的初始化顺序问题

# 初始化位置

两种初始化时机：
* main函数之前初始化：
  * 全局变量，也包括命令空间作用域变量
  * 文件域的静态变量
  * 类的静态成员变量
* 第一次调用时初始化：
  * 局部静态变量

c跟c++中局部变量的区别：
* c中时编译器便初始化跟分配内存的，因此只能用常量进行初始化
* c++中时第一次执行的时候初始化，这是因为c++引入了对象的概念，无法简单分配内存

# 动态/静态初始化

初始化分为静态初始化和动态初始化

## 静态初始化
针对于用常量来对全局变量进行初始化的情况，这个变量需要是内置类型

```bash
int a = 10;
```
例如上述代码，在编译器根据是否初始化放在bss或者data段

## 动态初始化

动态初始化发生在程序运行时，在main函数之前，不包含局部静态变量
包含如下几个类别：
* 1.需要经过函数初始化的，例如int a = fun();
* 2.复杂类型的初始化，需要调用构造函数的

静态初始化先于动态初始化发生

# 跨编译单元的静态存储器变量

不同编译单元的全局变量，初始化的先后顺序不确定，因此对于不同编译单元的全局变量互相引用的情况应该避免


一般在函数中声明局部静态变量，第一次调用时初始化，可以不受跨跨编译单元影响
